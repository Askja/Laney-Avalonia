# Laney 2

Laney 2 - кроссплатформенный неофициальный клиент ВКонтакте, написанный на .NET и [Avalonia](https://github.com/AvaloniaUI/Avalonia).

Этот репозиторий - форк оригинального проекта [Elorucov/Laney-Avalonia](https://github.com/Elorucov/Laney-Avalonia). Форк сохраняет идею Laney как легкого desktop-клиента VK, но добавляет локальные power-user функции, расширенные настройки, приватные сценарии, оптимизации производительности и нормальный рабочий набор для ежедневного использования.

Laney не пытается быть браузером в костюме приложения. Главные цели клиента: быстрый запуск, умеренное потребление памяти, предсказуемый интерфейс, локальный контроль данных и минимум лишнего визуального шума.

## Статус форка

Форк Askja развивает Laney 2 поверх upstream-версии:

- upstream: [Elorucov/Laney-Avalonia](https://github.com/Elorucov/Laney-Avalonia);
- fork: [Askja/Laney-Avalonia](https://github.com/Askja/Laney-Avalonia);
- стек: .NET, Avalonia, VK API, PowerShell build scripts;
- целевые платформы: Windows, Linux и macOS, насколько это позволяет текущая инфраструктура сборки;
- основной фокус: стабильность UI, производительность, локальные функции, приватность, кастомизация и удобство чатов.

Ограничение честное и скучное: если VK не отдает возможность через публичный API, Laney не будет обещать магию из воздуха. Часть функций официального клиента недоступна сторонним приложениям или требует отдельных web-сценариев.

## Авторизация

Доступны два сценария входа:

- OAuth через VK;
- вход по логину и паролю.

Приложение хранит локальные данные в пользовательской папке профиля. Для отдельных аккаунтов можно запускать Laney с параметром `-ldp=<path>` и держать несколько независимых профилей без ручной чистки токенов и кэша.

`not_robot_captcha` у VK требует WebView или браузерный сценарий. Если VK включает антибот-проверку, клиент может попросить пройти авторизацию через web-flow. Это не баг, это реальность VK API, чтоб его.

## Laney E2E

Laney E2E - это Laney-only шифрование поверх обычных VK-сообщений:

- passphrase-профили;
- X25519 handshake;
- fingerprint/SAS;
- auto-encrypt;
- encrypted payload;
- E2E-документы;
- trusted-device backup.

Официальный VK-клиент такой payload не расшифрует. Это локальная совместимость между клиентами Laney, а не новая серверная функция VK.

## Производительность

В форке закрыты задачи из UI performance-плана:

- снижена лишняя перерисовка списка сообщений;
- добавлены батчевые коллекции для тяжелых UI-обновлений;
- оптимизированы загрузка изображений и bitmap cache;
- добавлены ограничения памяти для медиа;
- выделены настройки lazy-load вложений;
- добавлены настройки анимаций стикеров;
- оптимизированы сетевые очереди;
- добавлены performance budget panel и chat health panel;
- добавлены локальные индексы поиска;
- улучшена работа с большими чатами и вложениями.

Главная идея: приложение должно оставаться отзывчивым при больших диалогах, медиа и активном Long Poll, а не превращаться в тест терпения.

## Локальная папка данных

Laney хранит настройки, токены, кэш, логи и локальные данные в профиле пользователя:

- Windows: `C:\Users\<User>\AppData\Local\ELOR\Laney\`;
- macOS: `/Users/<User>/Library/Application Support/ELOR/Laney`;
- Linux: `$HOME/.local/share/ELOR/Laney`.

Папку можно переопределить:

```bash
laney.exe -ldp=D:\custom\folder\for\account
```

Так можно держать несколько аккаунтов и запускать отдельные экземпляры Laney с разными локальными профилями. Повторный логин после каждой сборки не требуется, если профиль данных не удаляется и путь `-ldp` остается прежним.

## Демо-режим

В репозитории есть файл `demo.json`. Если скопировать его в локальную папку данных, Laney запустится в demo-режиме.

В demo-режиме показываются только чаты и сообщения из `demo.json`, а большинство сетевых функций недоступны. Это удобно для UI-разработки, проверки тем и тестирования без реального аккаунта.

Описание формата: [docs/demomode.md](docs/demomode.md).

## Сборка

Скрипт сборки находится в папке `L2`:

```powershell
pwsh ./build_aot.ps1
```

На Windows можно запускать так:

```powershell
pwsh .\build_aot.ps1
```

Скрипт выполняет release publish и выводит путь к собранной программе. Для Windows и macOS собираются x86-64 и arm64-версии, для Linux - x86-64. 32-битные сборки не поддерживаются.

Скрипт собирает бинарник только для той ОС, на которой запущен.

Канал сборки задается параметром `channel`:

```powershell
pwsh ./build_aot.ps1 -channel RELEASE
pwsh ./build_aot.ps1 -channel BETA
```

> [!WARNING]
> На данный момент скрипт сборки может работать некорректно на macOS.

## Платформенные требования

### Windows

Основная desktop-платформа для форка. Поддерживается кастомная рамка окна, запись голосовых и системные уведомления.

### Linux

Нужен `libvlc`:

```bash
sudo apt install libvlc-dev
sudo dnf install vlc-devel
```

Если пакет разработки не помог, установите сам VLC.

### macOS

Приложение не подписано. После установки может потребоваться снять quarantine-флаг:

```bash
xattr -r -d com.apple.quarantine /Applications/Laney.app
```

## CI/CD

При пуше в ветки `beta` и `release` автоматически запускается GitHub Action, который собирает и публикует сборки в Releases.

Перед релизным пушем нужно обновить версию:

- `L2/L2.csproj`;
- `MacOS_layout/Contents/Info.plist`.

## Ограничения

Laney использует публичные и доступные сценарии VK API. Поэтому есть ограничения:

- звонки зависят от доступности API и не обещаются как полноценная замена официальному клиенту;
- часть business-функций VK может быть недоступна;
- часть антибот-сценариев требует WebView;
- official-only возможности будут добавляться только если их можно реализовать легально и устойчиво;
- E2E работает только между клиентами Laney;
- локальные функции приватности не меняют историю на сервере VK, если VK API этого не поддерживает.
